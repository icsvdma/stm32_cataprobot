# 要件定義書
## デュアルMCUモーター制御システム（ESP32 + STM32）

| 項目 | 内容 |
|---|---|
| ドキュメントバージョン | 1.0 |
| 作成日 | 2026-03-19 |
| ステータス | ドラフト |
| MCU1 | ESP32-WROOM-32D (U4) |
| MCU2 | STM32G473RCTx (U5) |

---

## 1. システム概要

### 1.1 目的
Bluetooth コントローラーからの操作指令を ESP32 で受信し、UART/SPI 経由で
STM32 に転送してモーター駆動・センサ取得・LED 制御を行う
デュアル MCU 構成のモーター制御システムを構築する。

### 1.2 システム構成図

```
                        Bluetooth
                           │
                    ┌──────┴──────┐
                    │   ESP32     │
                    │ WROOM-32D  │
                    │             │
                    │ ・BT受信    │
  Battery ─────ADC──┤ ・電源管理  │
                    │ ・通信制御  │
                    └──┬────┬─────┘
                  UART2│    │SPI1
                    ┌──┴────┴─────┐
                    │   STM32     │
                    │ G473RCTx    │
                    │             │
                    │ ・モーター  ├──→ TB6575 (DC)
                    │ ・ステッパ  ├──→ TB6608 (Stepper)
                    │ ・センサ    ├──→ MPU6050 (I2C)
                    │ ・LED制御   ├──→ Headlight LED
                    └─────────────┘
```

### 1.3 役割分担

| MCU | 役割 | 理由 |
|-----|------|------|
| ESP32 | 通信・監視・電源管理（上位層） | Bluetooth/WiFi 内蔵、低消費電力モード対応 |
| STM32 | リアルタイム制御（下位層） | 高精度タイマ、豊富なペリフェラル、170MHz 動作 |

## 2. 機能要件

### 2.1 ESP32 機能一覧

#### F-E01: バッテリー監視（ADC）

| 項目 | 仕様 |
|------|------|
| 機能概要 | バッテリー電圧を ADC で定期取得し、残量を推定する |
| 使用ペリフェラル | ADC1（12bit） |
| 入力ピン | 回路図に準拠（分圧抵抗経由） |
| サンプリング周期 | 1 秒間隔（通常時）/ 100ms 間隔（低電圧警告時） |
| 電圧レンジ | 0V 〜 12V（分圧後 0 〜 3.3V） |

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-E01-01 | バッテリー電圧を ±0.1V 精度で取得できること | 必須 |
| F-E01-02 | 電圧値からバッテリー残量(%)を算出できること | 必須 |
| F-E01-03 | 低電圧閾値（設定可能）を下回った場合に警告を発すること | 必須 |
| F-E01-04 | 危険電圧閾値を下回った場合に自動シャットダウンを開始すること | 必須 |
| F-E01-05 | 電圧履歴を直近 60 件（1 分間）保持すること | 推奨 |
| F-E01-06 | BT 接続中のコントローラーへ残量を定期通知すること | 推奨 |

```
バッテリー電圧レベル定義:
  FULL     : >= 12.0V (100%)
  NORMAL   : >= 10.5V ( 50%)
  LOW      :  < 10.5V ( 20%)  → 警告LED点滅 + BT通知
  CRITICAL :  <  9.5V (  5%)  → 自動シャットダウン開始
  ※ 閾値は設定パラメータとして変更可能とする
```

#### F-E02: メイン電源保持（GPIO_POW_ON）

| 項目 | 仕様 |
|------|------|
| 機能概要 | 起動後に自身で電源保持信号を維持し、安全なシャットダウンシーケンスを管理する |
| 制御ピン | GPIO_POW_ON（GPIO25 付近、回路図に準拠） |
| 論理 | HIGH = 電源保持 / LOW = 電源遮断 |

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-E02-01 | 起動後 100ms 以内に GPIO_POW_ON を HIGH にすること | 必須 |
| F-E02-02 | シャットダウン時は STM32 への停止通知→モーター停止確認→電源遮断の順序を守ること | 必須 |
| F-E02-03 | シャットダウンシーケンスのタイムアウトは 3 秒とすること | 必須 |
| F-E02-04 | バッテリー危険電圧時に自動シャットダウンを実行すること | 必須 |
| F-E02-05 | コントローラーからの電源OFF指令に対応すること | 推奨 |

```
シャットダウンシーケンス:
  1. ESP32: STM32へシャットダウン通知送信 (UART)
  2. STM32: 全モーター停止 → LED消灯 → ACK返送
  3. ESP32: ACK受信確認（タイムアウト3秒）
  4. ESP32: 状態保存（必要に応じて）
  5. ESP32: GPIO_POW_ON = LOW → 電源遮断
```

#### F-E03: UART 通信（ESP32 ↔ STM32）

| 項目 | 仕様 |
|------|------|
| 機能概要 | コマンド・ステータスの送受信を行うメイン通信路 |
| 使用ペリフェラル | UART2 (ESP32: GPIO16/17, STM32: PA2/PA3) |
| ボーレート | 115200bps |
| データ形式 | 8N1 |
| 直列抵抗 | 22Ω（回路図準拠） |

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-E03-01 | パケットプロトコルによる構造化通信を行うこと | 必須 |
| F-E03-02 | チェックサム（XOR）によるエラー検出を行うこと | 必須 |
| F-E03-03 | 送信後 200ms 以内に ACK/NACK を受信すること | 必須 |
| F-E03-04 | ACK 未受信時は最大 3 回リトライすること | 必須 |
| F-E03-05 | 通信エラー率を記録し、異常検知に利用すること | 推奨 |

#### F-E04: SPI 通信（ESP32 ↔ STM32）

| 項目 | 仕様 |
|------|------|
| 機能概要 | 高速データ転送（センサデータ・バルクデータ） |
| 使用ペリフェラル | SPI1 (ESP32: GPIO12-15, STM32: PA4-7) |
| 役割 | ESP32 = Master / STM32 = Slave |
| クロック | 最大 10MHz（初期は 1MHz で検証） |
| 直列抵抗 | 22Ω（回路図準拠） |

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-E04-01 | MPU6050 のセンサデータを SPI 経由で高速取得できること | 推奨 |
| F-E04-02 | UART 通信障害時の冗長通信路として機能すること | 推奨 |
| F-E04-03 | UART と SPI の同時使用時に競合が発生しないこと | 必須 |

#### F-E05: Bluetooth コントローラー接続

| 項目 | 仕様 |
|------|------|
| 機能概要 | BT コントローラー（ゲームパッド等）からの操作入力を受け付ける |
| プロトコル | Bluetooth Classic SPP または BLE HID |
| 対応デバイス | PS4/PS5 コントローラー、汎用 BLE ゲームパッド |

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-E05-01 | BT コントローラーとペアリング・接続ができること | 必須 |
| F-E05-02 | アナログスティック値をモーター速度に変換できること | 必須 |
| F-E05-03 | ボタン入力をコマンド（停止・LED 切替等）に割当できること | 必須 |
| F-E05-04 | 接続切断を 500ms 以内に検知し、安全停止すること | 必須 |
| F-E05-05 | 再接続を自動的に試行すること（タイムアウト 30 秒） | 推奨 |
| F-E05-06 | コントローラーのバッテリー残量を取得できること（BLE HID） | 任意 |
| F-E05-07 | 入力のデッドゾーン・感度カーブを設定可能とすること | 推奨 |

```
コントローラーマッピング例:
  左スティック Y軸  → DCモーター速度（前進/後退）
  左スティック X軸  → ステアリング（ステッピングモーター）
  R2トリガー       → アクセル（速度倍率）
  L2トリガー       → ブレーキ
  ×ボタン          → 緊急停止
  △ボタン          → ヘッドライト切替（OFF→LOW→HIGH→AUTO→パターン）
  ○ボタン          → ハザード点滅
  □ボタン          → ホーン（将来拡張）
  OPTIONSボタン    → システムシャットダウン（長押し3秒）
```

### 2.2 STM32 機能一覧

#### F-S01: DC モーター制御（TB6575）

| 項目 | 仕様 |
|------|------|
| 機能概要 | TB6575 モータードライバを PWM で制御し、DC モーターの回転方向・速度を制御する |
| 使用ペリフェラル | TIM3_CH3 (PB0=PWM), GPIO (PB1=FST, PB2=CW_CCW, PB3=FMAX) |
| フィードバック | PB4 = FGOUT（回転数パルス入力、TIM 入力キャプチャ） |

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-S01-01 | PWM duty 0〜100% で速度を制御できること | 必須 |
| F-S01-02 | CW/CCW 方向切替ができること | 必須 |
| F-S01-03 | FGOUT パルスから回転数(RPM)を算出できること | 必須 |
| F-S01-04 | 急激な方向転換時に減速→停止→逆転のシーケンスを実行すること | 必須 |
| F-S01-05 | 過電流/ストール検出時に自動停止すること | 推奨 |
| F-S01-06 | 加減速カーブ（台形制御）を適用できること | 推奨 |

#### F-S02: ステッピングモーター制御（TB6608）

| 項目 | 仕様 |
|------|------|
| 機能概要 | TB6608 ステッピングモータードライバで 2 軸のステッピングモーターを制御する |
| 制御ピン（A軸） | PB10=CK_A, PB12=CW_CCW_A, PB14=RESET_A |
| 制御ピン（B軸） | PB11=CK_B, PB13=CW_CCW_B, PB15=RESET_B |
| 出力監視 | 6608_MO_A, 6608_MO_B（10kΩプルアップ） |

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-S02-01 | 指定ステップ数の正確な位置決めができること | 必須 |
| F-S02-02 | A/B 軸を独立して制御できること | 必須 |
| F-S02-03 | 加減速プロファイル（台形駆動）を適用できること | 推奨 |
| F-S02-04 | MO 出力による脱調検出を行うこと | 推奨 |
| F-S02-05 | パルス生成はタイマ割込で行い、メインループをブロックしないこと | 必須 |
| F-S02-06 | 原点復帰（ホーミング）機能を有すること | 推奨 |

#### F-S03: 加速度センサ（MPU6050）

| 項目 | 仕様 |
|------|------|
| 機能概要 | MPU6050 から加速度・ジャイロ値を取得し、姿勢推定・転倒検知を行う |
| 通信 | I2C (I2C_SDA/I2C_SCL, 10kΩプルアップ) |
| I2C アドレス | 0x68 (AD0=LOW) or 0x69 (AD0=HIGH) |
| サンプリング | 100Hz（10ms 周期） |

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-S03-01 | 加速度 3 軸(X/Y/Z)を取得できること | 必須 |
| F-S03-02 | ジャイロ 3 軸(X/Y/Z)を取得できること | 必須 |
| F-S03-03 | 温度データを取得できること | 任意 |
| F-S03-04 | 転倒検知（閾値超過）時に緊急停止を発行すること | 必須 |
| F-S03-05 | 相補フィルタまたは DMP による姿勢角算出を行うこと | 推奨 |
| F-S03-06 | センサデータを ESP32 へ定期的に送信できること | 推奨 |
| F-S03-07 | キャリブレーション機能（オフセット補正）を有すること | 推奨 |

#### F-S04: ヘッドライト LED 制御

| 項目 | 仕様 |
|------|------|
| 機能概要 | 前面 LED（左右）の点灯・調光・スマートライティングを制御する |
| 制御ピン | PA0 = FRONT_LED_R, PA1 = FRONT_LED_L（100Ω直列） |
| 制御方式 | TIM PWM による調光 |

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| F-S04-01 | ON/OFF 制御ができること | 必須 |
| F-S04-02 | PWM 調光（0〜100%）ができること | 必須 |
| F-S04-03 | 以下のライトモードを実装すること（詳細後述） | 必須 |
| F-S04-04 | コントローラーのボタンでモード切替ができること | 必須 |
| F-S04-05 | モード切替時にスムーズなフェード遷移を行うこと | 推奨 |

##### ヘッドライト スマートライティングモード一覧

| モードID | モード名 | 動作 | 優先度 |
|----------|----------|------|--------|
| LM_OFF | 消灯 | 両LED消灯 | 必須 |
| LM_LOW | ロービーム | 両LED 30%点灯 | 必須 |
| LM_HIGH | ハイビーム | 両LED 100%点灯 | 必須 |
| LM_AUTO | オートライト | MPU6050の姿勢角に応じて自動調光 | 推奨 |
| LM_ADAPTIVE | アダプティブ | モーター速度に連動 | 推奨 |
| LM_BRAKE | ブレーキライト | 減速検知時に自動で全灯 | 必須 |
| LM_HAZARD | ハザード | 左右同時点滅（500ms周期） | 必須 |
| LM_TURN_L | 左ウインカー | 左LED点滅 / 右LED通常 | 推奨 |
| LM_TURN_R | 右ウインカー | 右LED点滅 / 左LED通常 | 推奨 |
| LM_BREATH | ブリージング | 正弦波で明滅 | 任意 |
| LM_STROBE | ストロボ | 高速点滅（100ms周期） | 任意 |
| LM_WELCOME | ウェルカム | 起動時の演出パターン | 任意 |
| LM_BATTERY | バッテリー表示 | 点滅回数でバッテリー残量通知 | 推奨 |

```
モード遷移（コントローラー △ボタン短押し）:
  OFF → LOW → HIGH → AUTO → ADAPTIVE → OFF ...

自動割込モード（優先的に発動、元のモードに自動復帰）:
  ・LM_BRAKE   : 減速検知で自動発動 → 加速再開で復帰
  ・LM_HAZARD  : ○ボタンでトグル
  ・LM_WELCOME : 電源ON時に1回だけ自動実行
  ・LM_BATTERY : バッテリー低下時に通常モードに重畳
```

### 2.3 デバッグ LED 仕様

#### F-D01: ESP32 デバッグ LED

| LED | GPIO | 機能 | 点灯パターン |
|-----|------|------|-------------|
| D7 | IO22付近 | 電源状態 | ON=電源正常 / OFF=シャットダウン中 |

| D10 | IO26付近 | SPI通信 | 送受信時にトグル |

| D13 | IO33付近 | ユーザー定義 | 開発時のデバッグ用 |

```
起動シーケンス表示:
  電源ON → D7〜D13が順番に点灯（ナイトライダー風）→ 全消灯 → 通常表示
  所要時間: 約1秒
```

#### F-D02: STM32 デバッグ LED（FRONT_LED 兼用）

| 状態 | FRONT_LED_R (PA0) | FRONT_LED_L (PA1) | 意味 |
|------|-------|-------|------|
| 起動中 | 交互点滅(2Hz) | 交互点滅(2Hz) | 初期化中 |
| 正常動作 | OFF or ライトモード | OFF or ライトモード | 通常 |
| UART通信エラー | 4Hz点滅 | OFF | ESP32との通信異常 |
| I2Cエラー | OFF | 4Hz点滅 | MPU6050通信異常 |

| 緊急停止中 | 高速交互点滅 | 高速交互点滅 | 全モーター停止状態 |

## 3. 通信仕様

### 3.1 UART パケットフォーマット

```
Byte:  [0]      [1]     [2]       [3..N]      [N+1]
       Header   CMD     Length    Payload     Checksum
       0xAA     1byte   1byte    0-32bytes   XOR
```

### 3.2 コマンド体系

| CMD | 方向 | 名称 | Payload |
|-----|------|------|---------|
| 0x01 | ESP→STM | モーター速度設定 | [dir][speed_H][speed_L] |
| 0x02 | ESP→STM | モーター停止 | (なし) |
| 0x03 | ESP→STM | ステッピング制御 | [id][dir][steps_H][steps_L][speed] |
| 0x04 | ESP→STM | 緊急停止 | (なし) |
| 0x10 | ESP→STM | ステータス要求 | (なし) |
| 0x11 | ESP→STM | LED モード設定 | [mode_id][param1][param2] |
| 0x12 | ESP→STM | LED 輝度設定 | [led_id][brightness] |
| 0x20 | ESP→STM | センサデータ要求 | [sensor_type] |
| 0x30 | ESP→STM | シャットダウン通知 | (なし) |
| 0x80 | STM→ESP | ACK | (なし) |
| 0x81 | STM→ESP | NACK | [error_code] |
| 0x90 | STM→ESP | ステータス応答 | [motor_state][rpm_H][rpm_L][error_flags] |
| 0x91 | STM→ESP | センサデータ応答 | [ax_H][ax_L][ay_H]...[gz_L] (12bytes) |
| 0x92 | STM→ESP | 転倒検知通知 | [severity] |
| 0xF0 | STM→ESP | シャットダウンACK | (なし) |

### 3.3 エラーコード

| コード | 意味 |
|--------|------|
| 0x01 | 不明なコマンド |
| 0x02 | パラメータ範囲外 |
| 0x03 | モーター異常（過電流） |
| 0x04 | モーター異常（脱調） |
| 0x05 | センサ通信エラー |
| 0x06 | ビジー（処理中） |

## 4. 非機能要件

### 4.1 リアルタイム性

| 項目 | 要件 |
|------|------|
| コントローラー入力→モーター反映 | 50ms 以内 |
| 緊急停止（ボタン押下→モーター停止） | 10ms 以内 |
| モーター停止 | 20ms 以内 |
| BT 切断検知→安全停止 | 500ms 以内 |
| UART コマンド応答時間 | 200ms 以内 |

### 4.2 安全要件

| 要件ID | 要件 | 優先度 |
|--------|------|--------|
| NF-01 | BT 接続切断時は全モーター即時停止 | 必須 |
| NF-02 | 通信途絶（1秒以上コマンドなし）時は全モーター停止 | 必須 |
| NF-03 | バッテリー危険電圧時は安全シャットダウン | 必須 |
| NF-04 | 転倒検知時は全モーター即時停止 | 必須 |
| NF-05 | ウォッチドッグタイマを両 MCU で有効にすること | 必須 |
| NF-06 | モーター方向転換時の急反転防止（減速シーケンス） | 必須 |

## 5. 開発フェーズ

### Phase 1: 基盤確立（2週間）
- [ ] ESP32 / STM32 単体起動確認
- [ ] デバッグ LED 全点灯テスト（起動シーケンス）
- [ ] UART2 疎通テスト（エコーバック）
- [ ] パケットプロトコル実装・検証

### Phase 2: モーター制御（2週間）
- [ ] TB6575 PWM 制御（STM32 単体）
- [ ] TB6575 FGOUT 回転数取得
- [ ] TB6608 ステッピング制御（タイマ割込）
- [ ] ESP32 → UART → STM32 → モーター駆動の一気通貫

### Phase 3: センサ・LED（2週間）
- [ ] MPU6050 I2C 通信・データ取得
